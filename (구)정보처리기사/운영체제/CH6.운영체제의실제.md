# S1. UNIX운영체제
## 1. UNIX의 기본개념
- 주로 서버용 컴퓨터에서 사용
- 대부분 코드가 C언어로 기술된 대화식 시분할 운영체제

## 1.1 UNIX의 특징
- 대화식 운영체제로, 높은 이식성과 확장성이 있음
- 두 사람 이상의 사용자가 동시에 시스템 사용 가능
- 대부분 코드가 C언어로 기술됨
- 소스를 누구나 볼 수 있는 개방형시스템
- 파일소유자, 그룹 및 그 외 다른 사람들로부터 사용자를 구분하여 파일을 보호함
- 라이센스 비용 저렴
- 멀티태스킹(여러개프로그램동시수행), 멀티유저(여러사용자가동시사용) 기능 존재
- 계층적(트리구조) 파일 시스템
- 커널의 크기가 비교적 작아서 이식성이 뛰어남
- 커널 : 프로세스관리, 기억장치관리, 입출력관리 기능 수행
- 쉘 : 명령어 프로그램이 제공됨
- 사용자위주의 시스템 명령어 제공
- 주변장치를 파일과 동일하게 취급
- 파일 생성, 삭제, 보호 기능 가짐

## 1.2 UNIX 기본구성
### 커널(Kernel)
    - 하드웨어보호기능, 사용자서비스제공, 프로세스관리, 메모리관리, 네트워크(통신)관리, 입출력관리, 파일관리기능 등을 제공

### 쉘(Shell)
    - 사용자 명령의 입력을 받아 시스템 기능을 수행하는 명령해석기
    - 사용자와 시스템간의 인터페이스 담당

### 유틸리티
    - 운영체제가 지원하는 사용자프로그램
    - 문서편집기, 데이터베이스관리, 컴파일러, 네트워크(통신) 응용프로그램 등이 있음

## 2. UNIX의 파일시스템
### 1) Boot 블록
    부트스트랩시 사용되는 코드가 존재하며, 컴퓨터시스템 부팅 시 관련되 모든 정보를 가지고 있는 영역
### 2) 실린더블록
     [슈퍼블록]
     - 파일시스템을 관리하는 필수항목이 존재 
     - 파일시스템크기, I-node테이블크기, FreeBlock(사용가능 디스크 블록 개수 리스트) 등 파일시스템에 대한 종합적인 정보 저장

     [실린더그룹정보블록]
     - 사용블록의 정보, 통계적 정보 존재

     [I-node 테이블]
     - 파일의 정보를 기록한 파일정보테이블, 데이터가 저장된 블록의 시작주소를 확인가능한 블록

    [파일데이터블록]
     - 실제자료가 저장되어있는 공간
            
### 3) I-node(Index-node)
    - UNIX에서 각 파일에 대한 정보를 기억하고 있는 자료구조
    - 파일소유자의 식별번호, 파일크기, 파일의 최종수정시간, 파일링크 수(파일소유자의 수) 등의 내용을 저장하는 곳
    - UID : 사용자ID (파일소유자의 식별번호)
    - GID : 그룹ID (파일소유그룹의 식별번호)

### 파일과 디렉터리 소유
    읽기 r / 쓰기 w / 실행 x

# S2. PC 운영체제
## 1. MS-DOS
- 마이크로소프트 사에서 개발한 소규모 운영체제
- 단일 작업용 시스템 방식
- 한번에 한개의 프로그램만 실행 가능
- 한명의 사용자만 사용 가능
- Text 기반 인터페이스 방식
- 계층형 트리구조
- 블록단위파일사상 기법사용

## 2. MS-Windows
- GUI 그래픽지원 인터페이스
- 32비트 명령어형식 지원
- 가상기억장치 사용
- OLE(객체삽입기능) : 응용프로그램간의 자료 공유 가능
- 선점형 멀티태스킹
- Plug & Play 기능 제공
- VFAT : 가상 FAT사용 (ex. 파일명 255자)
- 네크워킹 기능있음
- 멀티미디어 지원
- Windows95 : 싱글유저, 멀티태스킹
- WindowsNT : 멀티유저, 멀티태스킹
