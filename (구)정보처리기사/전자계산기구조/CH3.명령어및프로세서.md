# S2. 명령어의 형식
## 1. 피연산자(operand address)개수에 따른 구분
0주소 : 스택

1주소 : 누산기

2주소 : 레지스터

3주소 : 메모리

## 1.1 0-주소 명령어
- 오퍼레이션(operation) 부분만 있고, 오퍼랜드(operand) 부분인 주소가 없는 명령어 형식
- **스택**에 있는 데이터 사용
- 모든 연산은 스택상에 있는 두 값을 묵시적으로 지정하여 POP해서 연산을 수행하고, 결과를 다시 스택상에 PUSH하기 때문에 원래의 데이터는 모두 잃어버리게 됨

## 1.2 1-주소 명령어
- 오퍼레이션 부분과, 1개의 오퍼랜드 부분으로 구성된 명령어 형식
- **누산기**에 있는 데이터 사용
- 연산 시 두개의 자료 중 하나는 묵시적으로 누산기를 지정하여 연산하고, 연산 결과 역시 누산기에 저장하는 방식
- 주기억장치 내의 데이터(주소에서 표현한 번지에 있는 데이터)와 누산기 내의 데이터로 연산이 이루어지며, 연산결과는 다시 누산기에 

## 1.3 2-주소 명령어
- 오퍼레이션 부분과, 2개의 오퍼랜드 부분으로 구성된 명령어 형식
- 연산 시 두개의 자료는 오퍼랜드가 지정한 2개의 자료를 대상으로 연산하고, 연산 결과는 주로 오퍼랜드1에 저장됨
- 연산 후 결과가 오퍼랜드1에 저장되므로, 오퍼랜드1의 위치에 있던 원래의 내용은 잃어버리게 됨
- 계산 결과를 시험(테스트)할 필요가 있을 때, 계산결과가 기억장치에 기억될 뿐 아니라, 중앙처리장치(CPU)에도 남아있어서 중앙처리장치 내에서 직접 시험이 가능함
- **레지스터**에 있는 데이터 사용

## 1.4 3-주소 명령어
- 오퍼레이션 부분과, 3개의 오퍼랜드 부분으로 구성된 명령어 형식
- 연산 시 두개의 자료는 오퍼랜드1과 오퍼랜드2에서 지정된 2개의 자료를 대상으로 연산하고, 연산결과는 오퍼랜드3에 저장됨
- 연산 후 결과는 오퍼랜드3 또는 오퍼랜드1에 저장되므로, 원래의 내용은 모두 보존할 수 있음
- 명령어의 길이가 길다
- **메모리**에 있는 데이터 사용

# S3. 주소지정방식
## 1. 주소지정방식의 빠르기
- **즉시(immediate) > 직접(direct) > 계산에의한(calculate) > 간접(indirect)**

## 2. 주소지정방식
## 2.1 즉시주소 (Immediate)
- 명령어의 주소 부분에 실제 데이터가 들어있는 방식
- 실제 데이터가 이미 들어있으므로, 가져오기 위해 메모리를 참조할 필요가 없음
- 그로인해 제일 수행속도가 빠르지만, 오퍼랜드에 실제데이터를 넣기 위해서는 명령어 길이에 제약이 따르는 단점이 있음

## 2.2 직접주소 (Direct)
- 명령어의 주소 부분에 데이터를 기억하고 있는 기억장소의 주소를 지정하는 방식
- 실제 데이터를 가져오기 위해서는, 메모리는 한번만 참조하면 됨 (주소가 가리키는 메모리번지에 한번 접근)
- 단순한 대신, 융통성이 떨어짐

## 2.3 간접주소 (Indirect)
- 명령어의 주소부분이 지정하는 메모리 번지에 실제데이터가 저장되는 방식
- 실제 데이터를 가져오기 위해 두 번 이상 참조해야함 (주소가 가리키는 메모리번지 접근 -> 메모리 번지가 가리키는 실제데이터 주소에 접근)
- 명령어의 길이가 짧고 제한되어 있어도, 이것을 이용하여 긴 주소를 찾아갈 수 있기 때문에 주소지정에 많은 융통성이 있음

## 2.4 계산에의한주소(Calculate)
오퍼랜드 부분에 있는 주소 값에 임의의 상수 값이나 레지스터에 있는 값을 더해서 유효주소를 결정하는 방법
## 2.4.1 인덱스주소지정
- 오퍼랜드 부분에 있는 주소값 + 인덱스 레지스터에 들어있는 값 합산
- 배열 인덱스 방식에 많이 사용
- ex) 오퍼랜드 : 100, 레지스터 50의 경우, 100+50인 150이 실제 데이터가 들어있는 기억장소의 위치
## 2.4.2 상대주소지정 (Relative)
- 오퍼랜드 부분에 있는 주소 값 = 프로그램카운터(PC)에 들어있는 값 합산
- 데이터영역의 배치와 프로그램영역의 배치가 유사한 경우 많이 사용
## 2.4.3 기준주소지정 (Base Register)
- 오퍼랜드 부분에 있는 주소 값 + 베이스레지스터에 들어있는 값 합산
- 프로그램의 재배치를 하는 경우에, 상대적인 변위 지정만으로 해당 위치를 지정하기 위해 많이 사용
- 다중 프로그래밍 기법에서 많이 사용
- *베이스레지스터 : 데이터블록(세그먼트)이나 서브프로그램의 시작주소를 기억

## 2.5 묵시적주소 (Implied)
- 0주소 명령어나 1주소 명령어 형식에서 스택이나 누산기의 위치를 지정할 때는, 명시적으로 지정하는 것이 아니라 묵시적으로 가리키게 됨

## 2.6 레지스터 주소
- 명령어의 오퍼랜드에 레지스터의 위치를 지정하는 방식
- 레지스터 직접모드 : 레지스터에 실제 데이터가 있는 방식
- 레지스터 간접모드 : 레지스터에 실제 데이터가 있는 기억장소의 주소가 들어있는

# S4. 프로세서
## 1. 프로세서의 종류
## 1.1 CISC (complex instruction set computer) 복합명령집합컴퓨터
- 초창기 컴퓨터 구조, 마이크로프로그램 제어방식 사용
- 명령어 크기 : 16~64비트, 명령어 개수 120~350개 정도로 많은편, 
- 소프트웨어적
- 소프트웨어 추가가 가능하기 때문에 명령어 개수가 많고 복잡. 대신에 호환성이 높고 가변적
- 메모리 참조연산을 많이함
## 1.2 RISC (reduced instruction set computer) 복잡하지 않은 명령집합컴퓨터
- 하드와이어드제어방식
- 주요명령들만 ROM(비휘발성)에 저장시켜 처리하고, 나머지 명령어들은 사용 시에 소프트웨어적으로 처리
- 명령어 크기 : 32비트로 고정크기, 명령어의 개수 100개 미만으로 적음
- 하드웨어적
- 수정이 불가하기 때문에 명령어 길이는 고정적이고 개수도 적지만, 속도는 빠름
- 레지스터 참조연산을 많이 함

## 2. 중앙처리장치(CPU)
## 2.1 중앙처리장치의 구성
1) 연산장치 (ALU, Arithmetic and Logic unit) 
- 산술연산 및 논리연산 등을 수행하는 장치
- 가산기, 누산기, 자리올림 플립폴롭,, 오버플로 체크 플립플롭, 보수기 등으로 구성
2) 제어장치 (CU, control unit)
- 주기억장치에 기억되어있는 명령어를 차례를 인출하여 해독하고, 해독한 결과에 따라 필요한 제어 신호를 각 장치에 보내는 역할
- PC, IR(명령어기억), 명령해독기, MBR(메모리내용저장), MAR(메모리접근), 순서제어기, 주소처리기, 신호발생장치 등으로 구성
3) 레지스터
- 입력된 자료, 상태자료, 중간계산자료, 최종결과자료 등을 보관하는 곳
## 2.2 중앙처리장치의 기능 (연산, 제어, 기억, 전달)
1) 연산기능 : ALU(연산장치)가 수행하는 기능. 데이터에 대해 산술 또는 논리연산 수행
2) 제어기능 : CU(제어장치)가 수행하는 기능. 제어 신호를 발생시켜 하드웨어 요소들을 통제
3) 기억기능 : 정보(데이터나주소)를 일시적으로 저장하기 위한 레지스터와, CPU나 프로그램의 상태를 나타내는데 쓰이는 플래그들에 의해 수행되는 기능
4) 전달기능 : 각 장치가 버스로 연결되어 정보를 주고받는 

## 3. 버스
결선의 수를 줄이기 위해 정보교환 시 버스 이용
## 3.1 버스의 형태
1) 주소버스 
- 메모리의 특정 위치를 지정하기 위한 단방향 회선
- 메모리의 최대용량과 관계 있음
2) 데이터버스 
- 중앙처리장치(CPU)와 주기억장치 사이에서 데이터를 주고받기 위한 양방향 회선
- CPU가 한번에 처리할 수 있는 비트 수
3) 제어버스 
- CPU나 메모리 또는 입출력 장치에 어떤 동작을 취하기 위해 제어 신호를 전달하는 단방향 회선

# S5. 레지스터
## 1. 레지스터
기억된 정보를 신속하게 접근할 수 있는 기억장소로, 중앙처리장치(CPU)안에 있는 임시기억장치
## 2. 레지스터의 종류 - 특수전담레지스터
용도가 미리 정해져있고 사용자의 접근이 불가능하여 불가시 레지스터라고도 함

    1) 프로그램 카운터(PC) : 다음 명령어의 주소 값 기억, 명령어의 수행 순서를 나타내는 레지스터
    2) 메모리번지레지스터(MAR) : 메모리에 접근하고자 할 때, 접근하고자 하는 곳의 주소 값을 기억하는 레지스터. 기억장치의 용량과 관련 
    3) 기억레지스터(MBR) : 주기억장치에서 읽은 정보나 저장시킬 정보를 임시적으로 기억하는 레지스터. 기억장치의 word크기와 관련
    4) 명령레지스터(IR) : 현재 수행하고 있는 명령어 기억

## 2. 레지스터의 종류 - 데이터레지스터(DR)
명령어 수행 시, 수행에 필요한 데이터를 기억시키는 레지스터
 
    1) 누산기(ACC) : 연산에 사용되는 데이터 및 연산결과를 임시적으로 기억
    2) 기억레지스터(MBR) : 주기억장치에서 읽은 정보나 저장시킬 정보를 임시적으로 기억
    3) 범용레지스터(GPR) : 함수연산기능을 수행하는 명령어에서 사용할 수 있도록 설계된 레지스터
    
## 2. 레지스터의 종류 - 주소레지스터(AR)    
주기억장치에 접근하기위한 주소의 일부 또는 전부를 기엇시키는 레지스터
  
    1) 프로그럄카운터(PC) : 다음 명령어의 주소 값 기억
    2) 메모리번지레지스터(MAR) : 메모리에 접근하고자 할 때, 접근하고자 하는 곳의 주소 값 기억
    3) 인덱스레지스터 : 주소변환 및 계산에 필요한 수 기억, 정보수정에 필요한 주소 값 기억, 배열의 시작주소와 같은 반복계수 기억 용도
    4) 베이스레지스터 : 하나의 서브프로그램 루틴 또는 데이터의 블록(세그먼트)에 대하여 기준이 되는 주소를 기억시키는 레지스터. 명령이 시작되는 최초번지 기억
    
## 2. 레지스터의 종류 - 상태레지스터
프로그램 제어를 위한 상태정보를 기억시키는데 필요한 레지스터

    1) 플래그레지스터 (=프로그램상태워드레지스터) 
    - 매 순간마다의 프로그램 수행상태를 포함하는 중앙처리의 상태를 나타내는 레지스터
    - 명령어 실행 중 어떤상황(부호체크, 오버플로우, 캐리발생여부 등) 발생에 대한 사항들을 각각 1비트로 세트시키는 레지스터
    2) 메이저상태레지스터(MSR) : CPU가 무엇을 하고 있는지 나타내는 레지스터
