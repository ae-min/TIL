# S1. 전용회선과 교환회선
## 1. 전용회선과 교환회선
## 1.1 전용회선(Leased Line)
- 두개이상의 단말기가 점대점(Point-to-Point)방식으로, 통신회선이 항상 고정적으로 연결되어있는 회선
- 통신장치와 통신회선이 많이 사용되므로 비경제적이지만, 고속의 데이터전송을 할 수 있고 전송오류가 적은 방법
- 전용회선으로 연결된 단말기의 개수가 N개이면, 통신회선의 수는 N(N-1)/2

      [전용회선의 특징]
      - 회선을 독점하여 사용
      - 회선설치 및 비용이 비싸며, 회선사용효율이 낮음
      - 통신량에 관계없이 고정비용을 내야함
      - 데이터 전송량이 많을 때 유리함
      - 전송속도 빠름 / 전송품질 좋음 / 보안성 좋음
      - 경로선택기술이 필요없음
      - 같은회선만 사용 가능
      - 통신범위가 제한적

## 1.2 교환회선(Switched Line)
- 송신측의 단말기에서 송신한 데이터를, 교환기가 식별하여 수신측에 있는 해당 단말기에 전송하는 방식
- 전용회선에 비해 고속전송과 전송데이터의 품질은 보장할 수 없지만, 통신장치와 회선비용을 줄일 수 있어서 경제적임

      [교환회선의 특징]
      - 회선을 여러개 컴퓨터에서 공유함
      - 회선설치 및 비용이 저렴하며, 회선사용효율이 좋음
      - 통신량에 따라 비용이 달라짐
      - 데이터전송량이 적을 떄 유리함
      - 전송속도 느림 / 전송품질이 나쁘고 오류가 발생함 / 보안성 취약함
      - 경로선택기술이 필요함
      - 다른회선 사용 가능
      - 통신범위가 넒음
## 1.3 데이터 교환방식 
- 회선교환방식 및 축적교환방식

## 2. 회선교환방식(Circuit Switching)
## 2.1 회선교환방식
- 경로가 확보되고 수신측에 응답이 있는 상태라면, 교환방식 중에서 전파지연이 가장 짧으며 실시간통신이 가능한 방식
- 하지만, 데이터전송 전에 경로를 확보하기 위해 긴 시간이 필요하기 때문에, 전체시간으로 볼 때는 많은 시간이 소요됨
## 2.2 회선교환방식의 특징
- 메시지가 전송되기 전에 발생지에서 목적지까지의 물리적인 통신회선연결이 선행되어야 함
- 데이터 전송 전에 먼저 통신망을 통한 연결이 필요함
- 전체 경로가 미리 확보되어야 함
- 일정한 데이터 전송률을 제공하므로, 동일한 전송속도가 유지됨
- 연결만되면 실시간통신이 가능
- 연결되면 통신회선은 공유되지 않고, 일대일 방식으로 데이터를 송수신하게 됨
- 통신회선이 독점되므로 통신비용이 비쌈
- 고정대역폭을 사용함 (고정대역폭 : 데이터전송속도가 항상 일정)
- 연결 시에 많은 시간이 필요함
- 코드와 속도가 다른 단말기간에는 통신이 불가능함
- 기억장치를 사용하지 않음
## 2.3 공간분할교환방식(SDS : Space Division Switching)
- 두 단말기 간의 경로가 공간적으로 분할된 방식
- 두 단말기 사이에 신호를 전송하는 교환기가 필요하고, 이 교환기를 거친 물리적인 경로가 설정되는 방식
- 즉, 교환기를 거치지만 두대의 단말기는 1:1로 연결되어있게 됨
### 2.3.1 단일단계교환방식
- N개의 입력과 N개의 출력으로 구성되어 있으며, N x N개의 교차점이 있는 교환기로 연결하는 방식
- 크로스매트릭스 교환방식 또는 크로스포인터 교환방식이라고도 함
### 2.3.2 다단계교환방식
- 단일단계교환방식의 단점을 보완한 방식
- N개의 입력 선을 분할하여 다음 그림과 같이 연결하게되면, 교환기 수가 줄고 경제적인 교환을 할 수가 있음 (1단계입력에서 3단계출력으로)

## 2.4 시간분할교환방식(TDS : Time Division Swith)
### 2.4.1 TDM버스교환
- 한 전송회선을 시간으로 분할하여 다중신호가 이 회선을 공유하는 방식
- 주로 동기식 TDM 버스교환을 사용
### 2.4.2 시간슬롯상호교환(TSI : Time Slot Interchange)
- 대부분의 시간분할교환방식이 채택하는 기술로, 전 2중동작을 위하여 한 쌍의 슬롯을 교환함으로써 시간슬롯 또는 채널의 동기화된 TDM열을 다룸
### 2.4.3 시간다중화교환(TMS : Time-Multiplex Swith)
- 시간슬롯상호교환 TSI장치는 제한된 수의 접속만을 제공
- 특히, 액세스 속도가 고정되었을 때 TSI의 크기가 커지면 지연시간은 더욱 커짐
- 이러한, TSI의 문제점을 해결하고 많은 양의 채널을 확보하기 위해 사용하는 방식

## 3. 축적교환방식(Store and Forward Switching)
## 3.1 축적교환방식
- 회선교환처럼 직접적으로 전기적인 연결은 없지만, 일시적으로 저장기능이 있어 네트워크 흐름이나 변화에 충분히 대처할 수 있는 장점이 있는 방식
- 통신회선을 공동으로 사용할 수 있으므로 경제적 통신 가능
- 전달되는 데이터가 축적교환기에 일시적으로 저장되었다가 전달되기 때문에, 저장시에 데이터를 변환(트랜잭션)시킬 수 있음
- 데이터변환 기능은 기종이 다른 어떠한 통신장비와도 통신이 가능하지만, 저장되기 때문에 전달속도가 느린 단점이 있음
- 전송할 데이터전체를 전송하는 메시지교환과, 전달속도를 향상시키기 위하여 데이터를 작은조각으로 분할해 전송하는 패킷교환방식이 있음
## 3.2 축적교환방식의 특징
- 기억장치를 사용
- 호출자와 피호출자가 동시에 운영상태에 있지 않아도 됨
- 데이터전송량이 폭주하는 혼란을 피할 수 있음
- 같은 내용의 메시지를 여러곳에 전송 가능
- 데이터의 손실을 막기위하여 부가적인 내용(번호, 날짜, 시간 등)을 추가 가능
- 코드와 속도가 다른 단말기 간에도 통신 가능
- 통신회선을 공유할 수 있기 때문에 통신비용이 저렴함
## 3.3 메시지교환방식
- 일반적인 데이터전송방법으로, 전송이 주 목적이기 때문에 전체 데이터를 한번에 전송함
- 한번에 전송을 하기 때문에 빠른 응답을 요구하는 전송에는 부적합
- 전송되는 전체 데이터는 전송순서를 기다리는 동안 교환기의 임시기억장치에 기억되었다가 순서가 되면 차례대로 전송됨
### 3.3.1 메시지교환방식의 특징
- 하나의 메시지단위로 **저장-전달방식(Store-and-Foward)**에 의해 데이터를 교환함
- 교환방식 중 전송지연시간이 가장 길다
- 응답시간이 느려 대화형 데이터 전송에는 부적절함
- 수신측이 준비가 안된 경우에도 지연 후 전송이 가능
- 전송코드와 속도가 다른 단말기끼리도 교환이 가능

## 3.4 패킷교환방식
- 메시지교환방식에서 데이터의 길이가 긴 메시지가 먼저 전송되고 있다면, 긴급을 요하는 짧은 메시지는 앞선 장문의 메시지가 모두 전송될 때까지 기다려야함
- 이러한 메시지교환방식의 문제점을 보완하여 빠른 응답시간을 제공하고, 통신회선의 효율적인 사용을 위하여 전송할 전체데이터를 일정한 크기로 나누어 전송하는 방식
- 이때 일정한 크기로 나누어진 데이터를 패킷이라고 함
### 3.4.1 패킷교환방식의 특징
- 축적(Store-and-Forward : 축적 후 전달)방식
- 메시지를 작은 데이터조각인 패킷으로 블록화함
- 데이터흐름이 많거나 교환기가 고장이 있어도 우회해서 전달될 수 있는 융통성이 존재함
- 모든 사용자간에 빠른 응답시간을 제공하기 위해 사용
- 트래픽 양이 적을 뿐만아니라, 많을 경우에도 적절하게 사용가능
- 채널과 포트의 통계적 다중화 기능을 제공
- 다수의 사용자간에 비대칭적 환경에서의 데이터전송을 원할하게 만들어줌 
- **가상회선방식과 데이터그램방식이 있음**
- 전송에 실패한 패킷의 경우 재전송 가능
- 패킷단위로 헤더를 추가하므로 패킷별 오버헤드가 발생함
- 실시간전송 불가

      [패킷교환망의 주요 기능]
      1) 패킷다중화 : 패킷들이 여러경로를 공유할 수 있도록 함
      2) 논리채널 : 가상회선교환채널이나 데이터그램교환채널을 설정
      3) 경로선택제어 : 최적의 경로를 설정
      4) 순서제어 : 패킷이 서로 다른 경로를 통하면, 패킷전송순서와 다르게 도착할 수 있으므로, 순서적으로 조립되도록 한다
      5) 트래픽제어 : 흐름제어, 체증제어, 교착상태 회피, 룩업상태해결 등을 지원
      6) 오류제어 : 패킷의 오류를 제거하든지 패킷을 삭제함

## 4. 패킷교환방식 중 가상회선방식과 데이터그램방식
## 4.1 가상회선패킷교환방식 (순서유지)
- 패킷을 전송하기 전에 미리 가상적인 경로를 확보하여 전송하는 방식
- 삽입에 대한 흐름이나 오류제어를 부 네트워크에서 지원하기 때문에 데이터그램 방식보다 오류가 적음
- 패킷의 송신순서와 수신순서가 바뀌지 않기 때문에 데이터그램방식에 비해 복잡하지 않음
- 경로를 확보하고 패킷을 전송(종속적)
- 호 설정, 초기설정 필요
- 목적지 주소를 초기 설정 시에 부여함
- 오류발생 시 교환기에서 지원함
- 패킷전송순서가 바뀌지 않음
- 전송량이 많을 때 유리함

## 4.2 데이터그램방식 (순서바뀔 수 있음, 재조립필요)
- 데이터전송 시 일정크기의 데이터 단위로 쪼개어, 특정 경로의 설정없이 전송되는 방식
- 패킷에 독립성을 부여하여 중간노드에 문제가 발생하여도 우회하여 목적지에 도착할 수 있는 방식
- 목적지가 같은 패킷이라도 다른 전송로로 진행할 수 있어서 매우 융통성있고, 소수의 패킷을 전송하는 경우에 유리함
- 언제든 순서가 뒤바뀔 수 있기 떄문에 수신측에서는 순서적으로 재조립해야 함
- 경로를 확보하지 않음(독립적)
- 호 설정, 초기설정 필요 없음
- 목적지 주소를 각각 부여
- 오류발생 시 교환기에서 삭제함
- 패킷전송순서가 언제든 바뀜
- 전송량이 적을 때 유리함

# S2. 경로선택과 트래픽제어
## 1. 경로선택(Routing, 라우팅)
- 송수신간에 중간서브넷(부 네트워크, 중간노드)을 거쳐 최적의 경로를 선택하는 기술
- 서브넷 자원의 이용을 최대화하여 평균패킷전송시간을 최소화시킴
- 데이터패킷을 출발지에서 목적지까지 이용가능한 전송로를 찾아본 후, 가장 효율적인 전송로를 선택하는 기술
## 2. 경로선택프로토콜
### 2.1 RIP (인터넷라우팅프로토콜, Routing Information Protocol)
- 라우팅테이블은 데이터그램 패킷을 통하여 모든 라우터에 전달됨
- 최대 hop을 15로 제한하므로, 15이상의 경우는 도달할 수 없는 네트워크를 의미함
- 홉hop : 패킷이 다음 네트워크 장비로 이동할 때마다 홉이 하나 발생
- **거리벡터알고리즘**을 사용하는 가장 단순한 라우팅 프로토콜
- 거리벡터알고리즘 : 행렬로 된 거리정보 값을 인접라우터와 주기적으로 교환하여 테이블을 갱신하고, 이를통해 경로를 선택함
- 매 30초마다 라우팅정보를 전달함
- 라우팅정보는 단지 180초 동안만 유효함
- hop의 수가 최대 15로 제한되어 있으므로 대규모 인터네트워킹에서는 RIP를 적용불가

      [기타 경로선택 프로토콜]
      
      1) OSPF (Open Shortest Path First)
      - RIP의 한계를 극복하기 위해 고안된 것
      - 네트워크의 변화가 있을 때에만 갱신함으로써, 효과적으로 사용할 수 있는 라우팅 프로토콜
      
      2) IGRP (Internet Gateway Routing Protocol)
      - BGP를 보완한 기술로 중간규모의 네트워크에서 사용하는 경로설정 프로토콜
      - 다양한 경로를 지정하여 최적의 경로를 설정 가능
      - IP네트워크에서만 사용가능하며 AS도 사용가능
      
      3) RTP (Reliable Transport Protocol)
      - IGRP를 개선한 EIGRP 패킷의 송수신을 위해 사용하는 프로토콜


### 2.2 ERP (외부라우팅프로토콜, Extend Routing Protocol)
- RIP만으로는 16개 이상이 되는 패킷의 교환경로를 가상으로 확보가 불가능함
- 따라서, RIP프로토콜을 사용하는 네트워크끼리 협력하여 패킷경로를 확보함

### 2.3 EGP (외부게이트웨이프로토콜, Extend Gateway Protocol)
- RIP만으로는 패킷망의 모든 경로를 가상으로 확보할 수 없음
- 따라서, TCP/IP(인터넷프로토콜)를 사용하는 네트워크와의 연합으로 패킷경로를 확보함

### 2.4 BGP (경계게이트웨이프로토콜, Border Gateway Protocol)
- AS(자율시스템)는 네트워크 내에서 게이트웨이 호스트들 간에 라우팅 정보를 교환하기 위한 프로토콜 (AS : 회사나 대학, 인터넷서비스업체 등의 경로선택 네트워크 시스템)
- BGP는 인터넷상의 게이트웨이 호스트들 간에 흔히 사용되는 프로토콜로, 관리자에 의해 독자적인 경로설정이 필요할 때 사용하는 프로토콜

## 3. 경로선택전략
## 3.1 비적응적방법
### 고정경로선택(Fixed Routing)
- 교환기마다 접속하려는 상대방에 미리 붙여둔 번호를 해석해서 진행경로를 선정하는 방식
- 가장 단순한 방법이나, 이미 정해진 경로로 진행하기 때문에 유연성이 없음
- 중간노드의 상태를 전혀 고려하지 않음
- 지나간 경로로 되돌아가는 루프가 없음
- 구현기술이 간단하기 때문에 경제적
### 범람경로선택(Flooding Routing, 플러딩)
- 입력된 경로를 제외하고 모든 경로에 패킷의 복사본이 전송되는 방법
- 신뢰성과 최단경로가 보장되지만, 복사된 패킷의 증가로 인해 체증이 유발될 수 있음
- 홉카운트를 이용해서 체증유발 방지 가능
- 홉카운트 : 송신측에서 전송될 때 전체 부 네트워크의 개수를 기억시키고, 복사될 때마다 홉 카운트를 1씩 감소시킴
- 중간 네트워크(서브넷) 정보를 이용하지 않음
## 3.2 적응적방법
- 비적응적방법을 보완한 방법으로, 중간노드의 체증이나 교착상태 등의 변화에 대한 정보를 얻어 능동적으로 경로를 선택하는 동적인 방법
### 국부적경로선택(Isolated Routing)
- 지역적으로 외부노드의 정보를 얻지 않고, 국부적인 노드의 독립적 판단에 의하여 결정되는 방법
- 거시적인 관점보다는 근시안적이기 때문에 체증유발 가능성이 높음
### 분산경로선택(Distribute Routing)
- 자신을 중심으로 연결되어 있는 모든 경로 선택기의 정보를 분석하여 광범위한 지역까지의 경로를 확보하는 방식
- RIP, ERP, EGP, BGP 등이 분산경로선택 프로토콜
- 그 중 RIP는 최대 16개의 경로선택기의 정보를 계속 파악 가능
- 국부적경로선택 방법은 인접한 경로 선택기의 정보만을 이용하기 때문에 체증유발의 가능성이 높을 수 있음
- 체증이 발생하는 문제점과 효율적인 경로 선택을 위해 분산경로선택 사용

## 4. 트래픽제어
- 네트워크내로 송수신되는 패킷의 수를 일정하게 조절하여 네트워크를 효율적으로 이용할 수 있도록 하는 기술
- 수신측의 버퍼량이 한계점을 초과하지 않도록 함
- 흐름제어, 체증(혼잡)제어, 교착상태 회피 등이 있음
## 4.1 흐름제어(FLow Control)
- 패킷교환방식에서 통신망 내 트래픽 제어의 원활한 흐름을 위해, 네트워크 내의 노드(컴퓨터 또는 교환기)와 노드 사이에 전송하는 패킷의 양이나 속도를 규제하는 기술
- 혹은, 수신기 버퍼의 오버플로우를 예방하기 위한 것으로, 데이터프레임의 전송률을 조정하는 것
- 두 노드 사이의 패킷 수를 조절하는 기술
### 슬라이딩윈도우 (Sliding Window)
- 한번에 여러개의 프레임을 전송할 경우 가장 효과적인 기법
- 송신측에서는 수신측으로부터 전송할 프레임의 개수(윈도우 크기)를 미리 지정받음
- 지정받은 개수만큼 송신측에서는 자율적으로 프레임의 개수를 전송할 수 있음
- 송신측에서는 지정받은 프레임의 개수만큼을 모두 전송하고나면 더이상 전송할 수 없게 됨
- 수신측에서는 전송할 프레임의 개수를 다시 지정하게 되면, 그때서야 다시 전송할 수 있게 됨
- 프레임의 개수가 증가하거나 감소하게 되는데, 수신측으로부터 이전에 송신한 프레임에 대한 긍정적인응답(ACK)이 왔을 때만 프레임의 개수가 증가하게 됨
- 수신 측에서 설정한 윈도우 크기만큼 송신 측에서 확인 응답(ACK) 없이 전송할 수 있게 하여 흐름을 동적으로 조절하는 제어 알고리즘
- 윈도우에 포함되는 모든 패킷을 전송하고, 전송이 확인되는 대로 윈도우를 옆으로 옮겨(slide) 다음 패킷들을 전송하는 방식

## 4.2 체증(혼잡)제어(Congestion Control)
- 네트워크 내에서 패킷의 대기지연이 너무 높아지게되어 트래픽이 붕괴되지 않도록, 네트워크 측면에서 패킷의 흐름을 제어하는 트래픽 제어기술
- 네트워크 상에 각 노드들의 패킷수신버퍼를 한계점 이하로 사용하도록 조절함
- 네트워크 상에 체증제어가 실패하게 되면, 패킷의 흐름이 전혀 없는 교착상태가 발생하여, 궁극적으로 인접한 모든 노드들까지 영향을 주게되는 룩업(Lock-up)상태가 됨
- 네트워크에서 패킷 수를 조절하는 기술

## 4.3 교착상태회피(Dead-Lock Aviodance)
- 흐름제어가 실패하면 체증이 유발되고, 패킷의 흐름이 정지되는 교착상태가 발생함
- 따라서, 같은 목적지를 갖는 패킷을 모두 할당하지 않도록 조절해야함
- 교착상태 발생 후 계속적으로 교착상태가 심화되면 룩업(Lock-up)상태가 발생하므로,
- 그 전에 교착상태에 있는 노드 중에 하나를 선택하여 패킷버퍼를 모두 폐기하면 룩업상태 방지 가능
- 이때 폐기되는 패킷들은 사용자에 의하여 재전송되어야 함

## 4.4 룩업상태(Lock-up)
- 패킷교환망에서 교환기를 여러사용자가 공유함으로써 패킷의 수가 급속히 증가하거나, 교환기 버퍼용량의 부족으로 교착상태가 발생함
- 이러한 부분적인 교착상태는 계속 확대되어 근접한 교환기까지도 영향을 주게되며, 이로 인하여 네트워크 전체의 패킷흐름이 멈추게 되는 현상을 말함

# S3.근거리네트워킹(LAN)
## 1. LAN의 이해
## 1.1 LAN
- 비교적 좁은지역에 분산배치된 컴퓨터와 프린터등의 단말기를 통신회선으로 연결하여 각종 정보를 교환할 수 있는 통신 네트워크
## 1.2 LAN의 특징
- 한정된 영역의 소규모 네트워크이기 때문에 경로선택을 필요로하지 않음
- 단일 기관의 소유 및 제한된 지역 내의 네트워크
- 광대역 전송매체의 사용으로 고속통신이 가능
- 오류발생률이 낮으며, 전송지연을 최소화할 수 있음
- 공유매체사용으로 경로선택 없이 매체에 연결된 모든 장치로 데이터 전송이 가능함
- 통신기기의 재배치와 확장성이 좋음
- 노드 값이 저렴하고, 어떤 종류의 통신시스템 기기와도 연결이 가능함
- LAN의 기본형태 : 스타형, 링형, 버스형, 계층형
## 1.3 LAN의 서비스영역에 따른구분
      1) MAN (Metropolitan Area Network)
      - 도시형통신망
      - LAN의 지역적인 협소성을 개선한 방법으로, 서비스범위는 대략 50km
      2) WAN (Word Area Network)
      - 지역적인 거리와 무관하게 같은 이해단체간을 연결한 망 형태
      - 각기 다른 LAN을 통합해야하는 어려움이 있지만, 망간의 폐쇄성 때문에 안전성이 뛰어나 많이 사용됨
      3) VAN (Value Area Network)
      - 통신회선을 직접 보유하거나 통신사업자의 회선을 임차하여 단순한 정보기능 이상의 부가가치를 부여하고, 
        음성 또는 데이터정보를 제공하는 광범위하고도 복합적인 통신서비스 형태
## 2.4 LAN의 통신방식에 따른 구분
### 베이스밴드방식
- 데이터전송 시 디지털데이터 신호를 변조하지 않고 직접 전송하는 방식으로, 대표적인 예는 이더넷이 있음
- 하나의 케이블에 대한 단일통신채널을 형성하여 데이터를 전송하기 때문에 모뎀이 필요없고, 경제적임
- 디지털신호이기 때문에 대부분 LAN은 베이스밴드방식
- 하지만, 공유할 수 있는 채널이 1개이기 때문에 서로 충돌을 방지하려고 CAMA/CD나 토큰링방식을 채택하기도 함
- 데이터전송방식 : CSMA/CD 또는 토큰링
- 단일채널/단일케이블
- 양방향통신
- 데이터전송에 응용됨
- 전송속도 : 1~10Mbps정도
- 소규모데이터전송에 적합
### 브로드밴드방식
- 부호화된 데이터를 아날로그 신호로 변조한 뒤, 제한된 주파수에게만 전송매체를 통해서 전송하는 방식
- 하나의 케이블에 다수의 통신채널을 형성하여, 데이터를 동시에 전송하는 방식
- 다수채널/단일케이블
- 데이터전송방식 : 토큰버스
- 디지털 신호를 아날로그신호로 변조하려면 모뎀이 필요하므로 비용적인 측면에서 부담되지만,
- 전송거리를 베이스밴드와는 달리 수십Km까지 늘릴 수 있는 장점이 있음
- 아날로그신호
- 채널이 20~30개
- 전송속도 : 150Mbps정도
- 단방향통신
- 음성, 영상전송에 응용됨
- 대규모멀티미디어전송에 적합

## 3. LAN의 기본형태
- 스타형, 링형, 버스형, 계층형
